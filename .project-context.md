# Project Context: Product Backlog Management

## 1. Project Identity
- **Name:** Product Backlog Management
- **Company:** PT Bina Pertiwi (Astra Group)
- **Primary Goal:** Monitoring Project & KPI Validation

## 2. Tech Stack Architecture
- **Framework:** Next.js (Note: Current local architecture uses Vite + React SPA)
- **Styling:** Tailwind CSS
- **UI Components:** Shadcn UI (Radix UI primitives)
- **Icons:** Lucide Icons
- **Deployment:** Vercel

## 3. Directory Map
- `/src/app/` - Main application routing (`routes.ts`) and root setup
- `/src/app/components/` - Domain-specific React components, pages, and layout
- `/src/app/components/ui/` - Reusable Shadcn UI component primitives
- `/src/app/data/` - Mock data and TypeScript interfaces for prototyping
- `/src/styles/` - Global stylesheets, theme definitions, and Tailwind CSS configuration

## 4. Design System Rules
- **Aesthetic:** Corporate Professional Style (Bina Pertiwi/Astra)
- **Visuals:** Clean, Professional layouts
- **Color Palette:** Corporate Blue & Grey accents with consistent semantic scaling
- **Responsiveness:** Mobile-first approach and fully responsive across breakpoints
- **Theming:** Full Light/Dark mode support via standard CSS variables & Tailwind classes

## 5. Staging Workflow & Operational Rules (MANDATORY)
1. **README Staging:** Sebelum melakukan operasional push, setiap perubahan (file yang diubah dan dampaknya terhadap antarmuka) **HARUS** dicatat di bagian `# Pembaruan & Riwayat Teknis` pada file `README.md`.
2. **Token Efficiency:** Gunakan file `.project-context.md` ini sebagai acuan (Source of Truth). DILARANG melakukan scanning seluruh direktori. Jika butuh file spesifik, baca hanya file tersebut.
3. **Final Push Protocol:** DILARANG melakukan `git push` ke repositori, kecuali terdapat instruksi spesifik "FINAL PUSH" dari pengguna.
4. **Baca file `README.md`** dan cek bagian `# Pembaruan & Riwayat Teknis` untuk melihat apa yang terakhir dikerjakan.
